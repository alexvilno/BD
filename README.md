ОТВЕТЫ НАХУЙ

1. Моделью данных называется формализованное описание структуры единиц информации и операций над ними в информационной системе.
2.  Структура - это описание объектов, хранимых в базе данных, и связей между ними. Ограничения целостности - это правила, которые определяют, какие значения можно хранить в базе данных, и какие операции разрешены или запрещены для этих значений. Операции - это действия, которые можно выполнять с данными в базе данных, такие как добавление, изменение и удаление объектов.
3. Иерархическая модель данных поддерживает структуру древовидного типа, где каждый объект может иметь один или несколько дочерних объектов, но при этом иметь только одного родителя. Объекты представляются с помощью записей, которые содержат элементы данных (поля) и ссылки на дочерние объекты. Связи между вершинами. одного уровня отсутствуют, а значит данные в иерархической структуре не равноправны – они жестко подчинены другим.  Примером может служить иерархия организации, где каждый отдел имеет своего руководителя.
4. Никак. Точнее: в иерархической модели связь многие ко многим не реализуется прямо - объект может иметь только одного родителя, и поэтому связь многие ко многим обычно заменяется двумя связями один-ко-многим. Короче говоря БЫЛО блять одно дерево – стало два. Пиздец избыточность. 
5. Иерархической модели данных присущи два важных внутренних ограничения. Первое из них состоит в том, что все типы связей далее будут функциональными, второе в том, что структура связей далее будет древовидной. Следствием этих ограничений является ряд особенностей структуризации данных. То есть: В силу функциональности связей запись может иметь не более одной родительской записи любого типа. В следствие ограничения на характер связи непосредственное представление связи типа M:N в иерархической модели невозможно. Эта тема описана в 4.
6. В иерархической модели данных поддерживаются базовые операции, такие как выборка, вставка, обновление, удаление, а также специализированные операции, такие как операция получения всех потомков узла. Пример: Выборка всех дочерних элементов заданного узла в иерархической структуре организации.
7. Порядок обхода дерева иерархии влияет на выполнение запросов, поскольку может повлиять на количество операций чтения и записи, необходимых для обработки запроса.
8. Преимущества иерархической модели данных:
    * Простота и эффективность хранения и обработки иерархических данных;
    * Высокая скорость выполнения запросов к данным;
    * Возможность эффективной обработки деревьев глубиной до 5 уровней.
	9. Недостатки и ограничения иерархической модели данных:
    * Ограничение на глубину дерева, что затрудняет обработку глубоких иерархий;
    * Ограниченность возможностей модели данных в отношении представления сложных связей между объектами;
    * Сложность поддержки целостности данных при изменении структуры дерева.
10. Примеры иерархических СУБД:
* IBM Information Management System (IMS);
* Computer Associates Integrated Data Store (CA-IDMS);
* Hierarchical Data Management System (HDMS);
* Univac DMS-1100.

ОТВЕТЫ НАХУЙ

1. В контексте информационных технологий, данные - это факты, числа, тексты, изображения, звуки и другие необработанные материалы, которые могут быть использованы для получения информации или знаний.
2. База данных (БД) - это организованная коллекция данных, которая используется для хранения, управления и доступа к информации. БД обычно организованы в виде таблиц, которые содержат структурированные данные, связанные между собой по определенным правилам.
3. Система управления базами данных (СУБД) - это программное обеспечение, которое позволяет создавать, управлять и обрабатывать данные в БД. СУБД предоставляет различные функции для управления данными, такие как добавление, удаление, изменение, поиск и сортировка.
4. СУБД не является просто надстройкой над операционной системой (ОС). Она обеспечивает свою собственную реализацию основных функций по обработке больших массивов данных, таких как поддержка транзакций, контроль целостности данных и оптимизация запросов.
5. Принцип минимальной и контролируемой избыточности означает, что каждый факт в БД должен храниться только один раз, чтобы избежать дублирования данных. В то же время, БД должна содержать достаточное количество дублирующих элементов, чтобы уменьшить время доступа к данным и ускорить выполнение запросов.
6. Реструктуризация и реорганизация базы данных - это процессы изменения структуры БД, которые могут включать в себя добавление, удаление или изменение таблиц, полей или связей между таблицами.
7. При использовании файловых систем для хранения и обработки данных, реструктуризация данных может привести к ряду недостатков, включая:
    * Необходимость перезаписи всех приложений, которые используют измененные данные.
    * Необходимость перекомпиляции приложений для обработки измененных данных.
    * Невозможность обеспечить целостность данных в случае сбоев или ошибок.
8. Недостатки файловых систем на примере возможностей по обработке файлов в традиционных системах программирования:
    * Файловые системы не обеспечивают целостность данных.
    * Файловые системы не могут обеспечить одновременный доступ к данным нескольким приложениям.
    * Файловые системы не обеспечивают возможность выполнения сложных запросов к данным.
    * Файловые системы не обеспечивают механизмы контроля доступа к данным.
9. Главная причина недостатков файловых систем при реструктуризации данных в случае, если одни и те же данные используются сразу несколькими приложениями, заключается в том, что файловые системы не предоставляют механизмов синхронизации доступа к данным. Это может привести к ошибкам и потере данных.
10. Преимущества, предоставляемые СУБД, связаны с улучшением производительности, надежности, безопасности и масштабируемости при обработке больших объемов данных. Однако, за эти преимущества приходится платить в виде более сложной архитектуры, сложности в управлении и обслуживании, а также более высокой стоимости.
11. При разработке приложений достаточно возможностей по обработке файлов в традиционных системах программирования, если данные не слишком большие или сложные, и нет необходимости обеспечивать одновременный доступ к данным нескольким приложениям. Однако, если данные имеют большой объем, сложную структуру или необходимо обеспечить одновременный доступ к данным нескольким приложениям, то будет рационально использовать СУБД.

ОТВЕТЫ НАХУЙ

1. Модель данных - это структурированный способ описания и организации данных, которые используются в информационных системах.
2. Реляционная модель данных поддерживает структуру данных в виде таблиц, которые состоят из строк (кортежей) и столбцов (атрибутов). Объекты представляются в виде строк таблицы, а взаимосвязи между объектами представляются в виде связей между таблицами.
3. Реляционное отношение - это множество кортежей (строк), где каждый кортеж представляет собой набор значений атрибутов, принадлежащих данному отношению. Декартово произведение - это операция, которая объединяет все возможные пары значений из двух отношений в единое отношение.
4. В реляционной модели данных взаимосвязь М:М представляется в виде дополнительной таблицы, которая связывает записи двух основных таблиц. Например, таблица "студенты" и таблица "курсы" могут быть связаны через дополнительную таблицу "регистрация на курсы", которая содержит записи об участии каждого студента в каждом курсе.
5. Свойства реляционного отношения:
    * Уникальность: каждый кортеж в отношении должен быть уникальным.
    * Неупорядоченность: порядок кортежей в отношении не имеет значения.
    * Атомарность: каждый атрибут в отношении должен быть атомарным, то есть не может содержать коллекции или другие структуры данных.
    * Однородность: каждый атрибут в отношении должен иметь одинаковый тип данных.
6. Реляционная модель данных поддерживает следующие внутренние и явные ограничения целостности:
    * Ограничение уникальности: каждое значение в атрибуте должно быть уникальным.
    * Ограничение ссылочной целостности: значения в одном атрибуте должны ссылаться на существующие значения в другом атрибуте.
    * Ограничение целостности домена: значения в атрибуте должны соответствовать определенному домену значений.
    * Ограничение целостности проверки: значения в атрибуте должны соответствовать определенному условию.
7. Реляционная алгебра - это набор операций, которые могут быть применены к реляционным отношениям. Операции реляционной алгебры используются для получения информации из БД, а также для манипулирования содержимым БД.
8. Реляционное исчисление - это формальный язык запросов, который используется для обращения к реляционным отношениям. Реляционное исчисление представляет собой декларативный язык, который описывает, какие данные необходимо извлечь из БД, а не как это сделать.
9. Примеры операций реляционной алгебры:
    * Выборка (select): выбирает из отношения только те кортежи, которые удовлетворяют определенному условию.
    * Проекция (project): выбирает из отношения только определенный набор атрибутов.
    * Объединение (union): создает новое отношение, которое содержит все кортежи из двух отношений.
    * Пересечение (intersect): создает новое отношение, которое содержит только те кортежи, которые присутствуют в обоих отношениях.
    * Разность (minus): создает новое отношение, которое содержит только те кортежи, которые присутствуют в первом отношении, но отсутствуют во втором.
10. Пример операции реляционного исчисления: выбрать все записи из таблицы "студенты", у которых оценка выше 4.0: { s | s ∈ студенты ∧ s.оценка > 4.0 }
11. Преимущества реляционной модели данных:
    * Простота использования и понимания.
    * Гибкость и расширяемость.
    * Независимость от физической структуры данных.
    * Высокий уровень безопасности и целостности данных.
12. Недостатки реляционной модели данных:
    * Сложность хранения и обработки сложных иерархических и многомерных данных.
    * Ограничения на производительность при работе с большими объемами данных.
    * Необходимость предварительного определения структуры данных.
    * Необходимость использования языков запросов для получения данных.

ОТВЕТЫ НАХУЙ

1. Модель данных - это структурированный способ описания и организации данных, которые используются в информационных системах.
2. Сетевая модель данных поддерживает структуру данных в виде набора записей (record) и связей между ними. Объекты представляются в виде записей, которые могут быть связаны друг с другом через отношения-связи.
3. Сингулярный набор в сетевой модели данных - это набор записей, в котором каждая запись может быть связана только с одной другой записью.
4. Область в сетевой модели данных - это набор связанных записей, которые могут быть доступны только через определенные записи-владельцы.
5. Набор (SET) в сетевой модели данных - это набор записей, которые связаны друг с другом через отношение-связь. Каждый набор имеет одного владельца, который может иметь несколько членов. Владелец набора является записью, которая содержит ссылку на первую запись-члена набора. Члены набора - это записи, которые связаны друг с другом через отношение-связь и ссылаются на владельца набора. Например, набор "заказы" может иметь владельцем запись "клиент", а членами набора будут записи "товары", связанные с заказами клиента.
6. В сетевой модели данных взаимосвязь М:М («многие ко многим») представляется в виде набора-отношения, в котором каждая запись может быть связана с несколькими записями другого набора. Например, набор "студенты" может быть связан с набором "курсы" через отношение-набор "регистрация на курсы", которое содержит записи о том, какие студенты записались на какие курсы, и наоборот.
7. Внутренние и явные ограничения целостности в сетевой модели данных:
    * Ограничение уникальности: каждый ключ должен быть уникальным.
    * Ограничение ссылочной целостности: ссылки могут указывать только на существующие записи.
    * Тип членства записей в наборе: определяет, как запись может быть связана с набором. Возможные типы членства:
        * Владелец (Owner): запись владеет набором и ссылается на первую запись в наборе.
        * Член (Member): запись является членом набора и ссылается на владельца набора.
        * Включение (Inclusion): запись включена в набор и ссылается на первую запись в наборе, но не является владельцем набора.
        * Удаление (Deletion): ссылка на запись в наборе была удалена, но запись сама по себе осталась в БД.
8. Операции над записями включают создание, обновление, удаление записей и поиск записей по ключу.
9. Операции над наборами включают создание и удаление наборов, добавление и удаление членов набора, поиск наборов и членов наборов.
10. Преимущества сетевой модели данных:
    * Позволяет представлять сложные отношения между данными.
    * Обеспечивает быстрый доступ к данным через связи-отношения.
    * Поддерживает эффективное использование памяти за счет минимизации дублирования данных.

    * Неэффективное использование памяти при повторном использовании данных в разных наборах.

ОТВЕТЫ НАХУЙ

1. Модель данных - это абстрактное представление структуры и организации данных в информационной системе, независимо от конкретной СУБД.
2. Основные компоненты модели данных:
    * Структура данных - определяет, как данные организованы и хранятся в БД.
    * Ограничения целостности - определяют правила, которые гарантируют, что данные в БД остаются корректными и целостными.
    * Операции - определяют, какие действия можно выполнять с данными в БД, например, добавление, изменение, удаление и поиск.
3. Универсальность модели данных означает, что она может быть применена к различным областям знаний и задачам, а также к различным типам СУБД.
4. Традиционные модели данных СУБД включают:
    * Иерархическая модель данных
    * Сетевая модель данных
    * Реляционная модель данных
5. Примеры семантических моделей данных СУБД:
    * RDF (Resource Description Framework)
    * OWL (Web Ontology Language)
    * Topic Maps
6. Примеры NoSQL моделей данных:
    * Колоночное хранилище данных (Column-Family)
    * Документоориентированная модель данных
    * Ключ-значение (Key-Value)
    * Графовая модель данных
7. Структуры данных, лежащие в основе моделей данных:
    * Иерархическая модель данных: древовидная структура данных.
    * Сетевая модель данных: графовая структура данных.
    * Реляционная модель данных: таблицы, состоящие из строк и столбцов.
8. Внутренние ограничения целостности - это ограничения, которые скрыты от пользователя, но которые обеспечивают корректность данных в БД. Явные ограничения целостности - это правила, которые задаются пользователем, чтобы гарантировать целостность данных.
9. Основные виды явных ограничений целостности:
    * Уникальность: каждое значение ключа должно быть уникальным.
    * Ссылочная целостность: значения внешнего ключа должны ссылаться на существующие записи в связанной таблице.
    * Правила проверки: определяют допустимые значения для поля.
10. Примеры внутренних ограничений целостности:
    * Иерархическая модель данных: каждый узел может иметь только одного родителя.
    * Сетевая модель данных: каждая запись может быть связана только с одной записью родительского типа.
    * Реляционная модель данных: каждая таблица имеет первичный ключ, который должен быть уникальным для каждой записи.
11. Ограничения целостности описываются в схеме БД. Они контролируются при выполнении операций, которые изменяют данные в БД.
12. Ограничения целостности, накладываемые на операции, определяют допустимые действия с данными в БД, например, запрет на удаление записи, если на нее есть ссылки из других таблиц. Ограничения целостности, накладываемые на данные, определяют допустимые значения для полей таблицы.
13. Операции навигационного типа позволяют перемещаться по связанным записям в БД, например, переходить от родительской записи к дочерним или наоборот.
14. Операции спецификационного типа позволяют выбирать определенные записи из БД, например, по заданным критериям поиска или сортировать записи в определенном порядке.
15. Примеры СУБД и их моделей данных:
    * Oracle: реляционная модель данных
    * MongoDB: документоориентированная модель данных
    * Neo4j: графовая модель данных
    * Microsoft Access: реляционная модель данных
    * Couchbase: ключ-значение модель данных.
